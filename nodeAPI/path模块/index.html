path模块

Author: 路耀广 Date: 九月 2, 2021  19:25:30

path模块

path.sep

提供特定平台的路径片段分隔符

path.basename( path [, ext ] )

path.basename会返回path的文件名部分，指定扩展名会被忽略

windows通常不区分大小写的方式来处理文件名(包括文件扩展名)，单此函数会区分

如果传入的path或者给定的ext不是字符串，则抛出TypeError。

path.delimiter

提供特定于平台的路径定界符

Window: ; POSIX: :

path.dirname(path)

返回文件路径

如果传入的path不是字符串，则抛出TypeError。

path.extname(path)

返回扩展名

path.format(pathObject)

从对象返回路径字符串，与**path.parse()**功能正好相反

pathObject参数为

  • dir 高优先级
  • root
  • base 高优先级
  • name
  • ext

当向pathObject提供属性时，存在一个属性优先于另一个属性的组合：

  • 如果提供了dir，则忽略root
  • 如果base存在，则忽略ext和name

path.isAbsolute(path)

判断是否为绝对路径

path.join([…paths])

使用特定平台的分隔符作为定界符，将所有给定的path片段连接在一起，然后生成规范化的路径

path.normalize(path)

规范化给定的path，解析”..”和”.”片段

path.parse(path)

根据path返回一个对象，表示path的重要元素。返回对象详情参见 path.foramt

path.relative(from, to)

据当前工作目录返回从 from 到 to 的相对路径。 如果 from 和 to 都解析为相同的路径（在分别调用 path.resolve() 之后），则返回零长度字符串。

如果零长度字符串作为 from 或 to 传入，则将使用当前工作目录而不是零长度字符串。

path.resolve([…paths])

path.resolve() 方法将路径或路径片段的序列解析为绝对路径。

给定的路径序列从右到左处理，每个后续的 path 会被追加到前面，直到构建绝对路径。 例如，给定路径片段的序列：/foo、/bar、baz，调用 path.resolve('/foo', '/bar', 'baz') 将返回 /bar/baz，因为 'baz' 不是绝对路径，而 '/bar' + '/' + 'baz' 是。

如果在处理完所有给定的 path 片段之后，还没有生成绝对路径，则使用当前工作目录。

生成的路径被规范化，并删除尾部斜杠（除非路径解析为根目录）。

零长度的 path 片段被忽略。

如果没有传入 path 片段，则 path.resolve() 将返回当前工作目录的绝对路径。

从右到左匹配到一个绝对路径地址就停止
